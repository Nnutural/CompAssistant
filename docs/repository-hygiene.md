# 仓库卫生说明

## 默认应提交

- 人工维护的源代码文件。
- 文档与 schema。
- `.env.example`、`.codex/config.toml` 这类样例配置。
- 技能定义与 agent 指南文件。
- 已纳入仓库工作流的 lockfile。

## 默认应忽略

- 密钥和本地 `.env` 文件。
- Python 虚拟环境。
- `node_modules` 与前端构建产物。
- 测试输出、覆盖率产物、缓存、日志和临时导出。
- 除非团队统一约定，否则应忽略机器本地 IDE 配置。

## 提交前检查清单

- 确认新增文档已清晰说明边界。
- 确认占位目录没有引入真实 import 接线。
- 确认样例配置文件不包含真实密钥。
- 确认本地产物已被忽略规则覆盖。
